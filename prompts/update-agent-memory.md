# Update Agent Memory

Use this when a task reveals a repeated rule, project boundary, command, or
workflow detail that future agents should know.

## Inputs

- What went wrong or was corrected:
- How many times it has happened:
- Permanent rule proposed:
- Files affected:

## Rules

- Update `AGENTS.md` for agent-operational rules.
- Update `CLAUDE.md` for architecture or project-specific development guidance.
- Update docs under `docs/` for product or design decisions.
- Keep memory concise and durable.
- Do not add one-off incident details as permanent rules.

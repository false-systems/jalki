# Python SDK

## Four methods, nothing else public
- `jalki.ask(question)` → AskResult
- `jalki.find(question)` → list[ProbeMatch]
- `jalki.deploy(function)` → ProbeHandle
- `jalki.stream(handle)` → AsyncIterator[Event]

## find() works offline
- No daemon, no network, no event loop required
- Uses embedded knowledge base — same JSON files as daemon
- `jalki.find("connection refused")` returns tcp_connect as first result
- `jalki.find("packet loss")` returns tcp_retransmit_skb as first result

## ask() graceful fallback
- If daemon not running: returns AskResult with kb_only=True
- interpretation and action are always non-empty strings
- Never raises an exception — always returns something useful

## Event shape is compact
- Events have: id, ts, probe, severity, outcome
- net field combines ip:port into single strings
- No FALSE Protocol bloat fields: no source, enrichment_state, entity_ids, correlation_keys

## Generated types
- types.py and protocol.py are GENERATED by jalki-sdk-meta
- They must have "GENERATED" and "DO NOT EDIT" in the header
- types.py defines: Severity, Outcome, Proto, Attachment as IntEnum
- types.py defines: Event, NetData, ProcData, EventFilter, ProbeMatch, ProbeHandle, AskResult as dataclass

## Behavioral parity
- Same method names across all SDK languages
- Same parameter names
- Same event shape
- Same error types

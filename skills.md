# jälki Agent Skill

You are working in the jälki repository — a programmable fentry/fexit framework for Linux kernel observability. Read this file before writing any code.

## What jälki is

jälki is to kernel functions what POLKU is to gRPC — a programmable framework. You implement one Rust trait, jälki handles BTF loading, fentry/fexit attachment, ring buffer management, self-filtering, serialization, and emission. FALSE Protocol Occurrences out.

```
your probe (Rust trait or Python decorator)
    ↓
jälki framework
    ↓
FALSE Protocol Occurrence JSON
    ↓
stdout / file / gRPC (POLKU) / SDK
```

The three built-in TCP probes are batteries-included. They are not what jälki *is*.

## Build Order — Always Follow This

```bash
# 1. eBPF first — always
cargo run -p xtask -- build-ebpf [--release]

# 2. Then userspace
cargo build [--release] -p jalki

# 3. Then SDK meta (if touching SDK)
cargo run -p jalki-sdk-meta -- --lang python --out jalki-sdk-python/src/jalki/

# 4. Then tests
cargo test --workspace
```

Never skip step 1. Userspace will compile but the daemon will fail at runtime with a missing eBPF object.

## Crate Structure


## The Rules

### Never edit generated files

These files are generated by `jalki-sdk-meta`. Never edit them by hand:
- `jalki-sdk-python/src/jalki/types.py`
- `jalki-sdk-python/src/jalki/protocol.py`
- Any file with `# GENERATED by jalki-sdk-meta` header

To change them: edit `jalki-sdk-meta/src/` and regenerate.

### Never modify the oracle to make tests pass

`eval/oracle/` is the ground truth. If an oracle test fails, fix the system or the knowledge base JSON. Never change the oracle test to match broken behavior.

```bash
# Run oracle
cargo test --manifest-path eval/oracle/Cargo.toml
# If this fails: fix knowledge/*.json or the system, not the test
```

### Never change jalki-common struct sizes without updating size tests

`TcpConnectEvent`, `TcpCloseEvent`, `TcpRetransmitEvent` have size tests that lock the BPF ABI. If you change them, update ALL size tests. If the size changes, the eBPF ring buffer reads will be wrong at runtime.

```rust
// These must stay correct after any struct change
#[test]
fn tcp_connect_event_size() {
    assert_eq!(mem::size_of::<TcpConnectEvent>(), 56);
}
```

### No .unwrap() in userspace code

Use `?` or explicit error handling. `unwrap()` in eBPF code is fine (it's necessarily unsafe). In userspace it will panic the daemon.

### No println! in library code

Use `tracing`. `println!` goes to stdout which is the event stream.

## Adding a New Probe — 3 Steps

This is the core operation. Do it exactly in this order.

### Step 1 — Define the event struct in jalki-common

```rust
// jalki-common/src/events.rs
#[repr(C)]
#[derive(Clone, Copy)]
pub struct MyEvent {
    pub timestamp_ns: u64,
    pub pid: u32,
    // ... your fields
    pub _pad: u32,  // always pad to 8-byte alignment
}

// mandatory size test — lock the BPF ABI
#[test]
fn test_my_event_size() {
    assert_eq!(core::mem::size_of::<MyEvent>(), 16);
}
```

Add `#[cfg(feature = "userspace")] unsafe impl aya::Pod for MyEvent {}`.

### Step 2 — Write the eBPF program in jalki-ebpf

```rust
// jalki-ebpf/src/my_probe.rs
#[fentry(function = "some_kernel_function")]  // or fexit
pub fn my_probe(ctx: FEntryContext) -> i32 {
    if is_self_filtered() { return 0; }
    // read fields via bpf_probe_read_kernel
    // write to ring buffer
    0
}
```

Wire up the ring buffer map in `jalki-ebpf/src/main.rs`.

### Step 3 — Implement the Probe trait in jalki

```rust
impl Probe for MyProbe {
    fn name(&self) -> &str { "my_probe" }
    fn program_name(&self) -> &str { "my_probe" }  // must match fn name in eBPF
    fn attachments(&self) -> &[Attachment] {
        &[Attachment::Fentry { function: "some_kernel_function" }]
    }
    fn ring_buffer_map(&self) -> &str { "MY_EVENTS" }
    fn to_occurrence(&self, raw: &[u8], cluster: &str) -> Result<Occurrence, ProbeError> {
        // convert raw bytes → FALSE Protocol Occurrence
    }
}
```

### Step 4 — Add knowledge base entry

Every new probe needs an entry in `knowledge/{layer}.json`. Follow the exact schema. The oracle will validate it on the next run.

Required fields: `function`, `attachment`, `event_type`, `layer`, `answers`, `keywords`, `fields` (each with `name`, `type`, `meaning`, `important`), `use_when`, `not_when`, `combine_with`, `interpretations` (each with `pattern`, `conclusion`, `severity`, `action`).

**Do not add KB entries you are not certain about. Wrong interpretations mislead agents.**

## fentry vs fexit — When to Use Which

**Use fexit when you need the return value.** fexit fires after the function returns. You get both input args AND the return value. Use for: tcp_connect (did it succeed? errno?), tcp_sendmsg (how many bytes?), inet_csk_accept (did accept succeed?).

**Use fentry when you only need entry state.** fentry fires before execution. Use for: tcp_retransmit_skb (you want to know it's happening, not what it returned).

Rule: if the question involves success/failure/errno → fexit. If the question is "did this happen" → fentry.

## The Wire Protocol

The daemon speaks binary frames + MessagePack over `/run/jalki/jalki.sock`.

```
[frame_len: u32 big-endian][msg_type: u8][flags: u8][msgpack payload]
frame_len = payload.len() + 2
```

Constants are in `jalki-sdk-meta/src/protocol.rs` — single source of truth. Do not hardcode message type bytes anywhere else.

STREAM_EVENT is a positional MessagePack array — no field names. Positions are `POS_*` constants. Do not add named fields to the wire format.

## The SDK

Four methods, no more. `ask`, `find`, `deploy`, `stream`.

`find` runs locally — KB embedded in SDK, no daemon needed, zero tokens.
`ask` is the magic method — find → deploy → collect → interpret server-side.
`stream` is stateless — no cursor, no resumption. For durable delivery use POLKU.

AHTI does not talk to jälki. AHTI consumes from POLKU.

```
kernel → jälki daemon → POLKU (gRPC) → AHTI
                      ↓
                   Unix socket
                      ↓
                   SDK / CLI / MCP
```

## Knowledge Base

The KB lives in `knowledge/*.json`. It is:
- compiled into the daemon binary at build time (`include_str!`)
- compiled into every SDK at build time
- validated by the oracle at test time

To add or change an entry: edit the JSON, run `cargo build` (compile error = malformed JSON), run `cargo test --manifest-path eval/oracle/Cargo.toml` (test failure = wrong semantics).

## Common Mistakes and How to Fix Them

### BPF verifier rejection
Most verifier errors come from: unbounded loops, accessing memory without `bpf_probe_read_kernel`, stack frame too large (>512 bytes), or UB checks. The xtask build uses `-Zub-checks=no` for this reason. If you get a verifier error, check offset reads are using `bpf_probe_read_kernel` and that loops have bounded iterations.

### Struct offset wrong on different kernels
Always use BTF-resolved offsets via `jalki-codegen/src/btf.rs`. Never hardcode struct offsets. The known offsets in comments (`// offset 0: skc_daddr`) are verified on kernel 6.x via pahole — other kernels may differ.

### dst_ip 0.0.0.0 on Cilium nodes
This is NOT a bug. Cilium transparent proxy rewrites the destination via eBPF before the kernel populates `skc_daddr`. Document it, don't try to fix it.

### src_port 0 on tcp_close
NOT a bug. Kernel clears `skc_num` before tcp_close returns. Use the tcp_connect event's src_port and correlate by 4-tuple. This is documented in CLAUDE.md and README.

### SDK conformance tests failing
Run them in order: no-daemon tests first, daemon tests second. No-daemon tests must always pass regardless of whether a daemon is running. If a daemon test fails, check the daemon is running the binary protocol (not the old ndjson shim).

### Oracle failing after KB change
Read the error carefully. The oracle tests semantics, not just schema. `case_014` means ESTABLISHED retransmit must say "network problem". `case_016` means ECONNREFUSED must say "nothing is listening". These are correct — fix the KB entry.

## FALSE Protocol Occurrence Shape

Every probe emits this. The SDK compact event is a subset of this.

```json
{
  "id": "01JWXYZ...",           // ULID
  "timestamp": "2026-...",      // RFC3339 nanoseconds
  "source": "jalki/tcp_connect",
  "type": "kernel.tcp.connect",
  "severity": "info",
  "outcome": "success",
  "cluster": "dev",
  "enrichment_state": "raw",
  "correlation_keys": ["10.0.0.1:54321->10.0.0.2:8080"],
  "network_data": { ... },
  "process_data": { ... },
  "labels": { "tcp_state": "ESTABLISHED" }
}
```

`correlation_keys` is the 4-tuple string. AHTI uses this to join jälki events with RAUTA and TAPIO events.

## Running Tests

```bash
# Fast validation — no eBPF hardware needed
cargo check --workspace
cargo test --workspace

# Oracle — ground truth contract tests
cargo test --manifest-path eval/oracle/Cargo.toml

# Single oracle case
cargo test --manifest-path eval/oracle/Cargo.toml -- case_014

cd jalki-sdk-python
pytest tests/test_conformance.py -m "not daemon" -v

```
